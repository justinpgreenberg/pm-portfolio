## How I Work

My operating sequence is consistent across zero-to-one and scale-stage work:

1. **Define the product thesis in a PRD**
   - User problem, business objective, constraints, and non-goals are explicit.
2. **Design a testable path to value**
   - Break the experience into value moments, then identify activation and conversion hypotheses.
3. **Instrument before launch**
   - Events, properties, and funnel definitions are set pre-release so decisions are evidence-based from day one.
4. **Run structured experiments**
   - Each experiment has a hypothesis, success metric, decision threshold, and follow-up action.
5. **Iterate on system bottlenecks**
   - Improve the weakest link first (match quality, onboarding friction, message relevance, etc.).
